Additional task
PImpl
Rework percolation.h to use the "PImpl" approach. This means the Percolation
structure definition will no longer be in the header file - it should be moved to the .c
file. Also, update unit tests for the first task if needed.
The PImpl (Pointer to Implementation) technique was described in the previous
course, Introduction to Programming. In short, it is a method in C and C++ that hides
implementation details by using a pointer to a private structure. This improves
encapsulation, reduces compilation dependencies, and makes the code more
modular, since internal changes do not affect the public interface. By keeping the
structure definition in the .c file, only functions can access or modify the data,
preventing direct access from outside the module.
Points: 1
